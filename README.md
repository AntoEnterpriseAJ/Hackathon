# Teacher Paperwork Assistant

AI-assisted toolkit for processing, validating, and comparing Romanian academic documents (Fisa Disciplinei / Plan de Invatamant). Built for a hackathon.

## Stack

- **Backend** — FastAPI + Anthropic Claude (Sonnet 3.5), PyMuPDF, pdfplumber, python-docx
- **Frontend** — Angular 19

## Repository Layout

| Path | Description |
| --- | --- |
| [backend/](backend/) | FastAPI app: parsing, validation, suggestion, drafting, and diffing endpoints |
| [frontend/](frontend/) | Angular UI (upload, guard editor, diff viewer) |

## Key Endpoints

Backend (`http://localhost:8000`):
- `POST /api/documents/parse` — extract structured data from PDF/image
- `POST /api/documents/validate` — validate against template + guards
- `POST /api/documents/suggest` — semantic fix suggestions
- `POST /api/documents/draft-fd` — draft a Fisa Disciplinei from a Plan
- `POST /api/documents/cross-validate` — check FD vs PI consistency
- `POST /api/documents/diff` — textual diff between two documents
- `POST /api/documents/diff-visual` — visual PDF diff (annotated PDFs)

## How It Works

Conceptually the system has four jobs: **get text out of a document**, **understand what kind of document it is**, **check it against rules**, and **compare versions**.

### 1. Ingestion & routing

An uploaded file (PDF, image, or DOCX) hits the backend and is dispatched based on what it actually is:

- **DOCX** → parsed structurally with `python-docx` (no AI needed).
- **Image** (`.png/.jpg`) → sent straight to the vision pipeline.
- **PDF** → goes through [pdf_router](backend/services/pdf_router.py), which samples text density. High density means it's a *text PDF* and we extract directly with PyMuPDF. Low density means it's a *scan* and we render pages to images at ~150 DPI and treat it like the image path.

This avoids paying for vision tokens on PDFs that already have selectable text.

### 2. Extraction

- **Text path** → [text_extractor](backend/services/text_extractor.py) pulls raw text, then Claude turns it into a structured `ExtractedDocument` (fields, tables, summary).
- **Image / scan path** → [scan_extractor](backend/services/scan_extractor.py) feeds pages to Claude's vision model, batched/paged for long PDFs.
- Results are cached in [parse_cache](backend/services/parse_cache.py) so re-uploads are cheap.

A [document_classifier](backend/services/document_classifier.py) labels the document as a **Fisa de Disciplina (FD)** or a **Plan de Invatamant (PI)** so the rest of the pipeline knows which schema and rules to apply. Bundles containing several FDs are split by [fd_bundle_splitter](backend/services/fd_bundle_splitter.py).

### 3. Validation & suggestions

For a known document type the backend builds a **template + schema + guard set**:

- **Template** = the canonical structure of the document type.
- **Guards** = declarative rules (numeric ranges, sums that must match, type checks, cross-field consistency). Defaults are generated by [template_drafts](backend/services/template_drafts.py); the user can tweak them in the UI.
- [template_validator](backend/services/template_validator.py) runs the guards and returns a list of violations.
- Cross-document checks ([cross_doc_validator](backend/services/cross_doc_validator.py), [numeric_consistency](backend/services/numeric_consistency.py), [bibliography_checker](backend/services/bibliography_checker.py)) verify that an FD is consistent with its parent PI (hours, ECTS, competencies, etc.).
- If there are violations, [template_suggester](backend/services/template_suggester.py) asks Claude to propose patches with confidence scores, which the user can apply back into the document.

### 4. Drafting

Given a Plan de Invatamant, [fd_drafter](backend/services/fd_drafter.py) can generate a first-draft Fisa Disciplinei for any course in it (Claude fills the template, [fd_docx_renderer](backend/services/fd_docx_renderer.py) writes it out as a `.docx`).

### 5. Diffing

For comparing two versions of the same document the backend ([documents.py](backend/routers/documents.py)) wires together:

- **Extractor** pulls per-page text with `pdfplumber`.
- **Parser** (FD or PI) splits the text into named sections.
- **Differ** ([diff_difflib_differ](backend/services/diff_difflib_differ.py)) does line-level diffs per section using `difflib`.
- **Analyzer** ([diff_regex_analyzer](backend/services/diff_regex_analyzer.py)) runs regex rules on top of the raw diff to flag *semantic* changes (hours changed, ECTS changed, bibliography changed, ...) instead of just "lines moved".
- **Visual differ** ([diff_visual_differ](backend/services/diff_visual_differ.py)) renders the two PDFs with red/green bounding boxes around removed/added regions and returns them base64-encoded.

The Angular frontend ([frontend/](frontend/)) wires all of this together: upload + guard editor on the home page, side-by-side + visual diff on the diff page.

## Quick Start

### Backend

```powershell
cd backend
python -m venv ..\.venv
..\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
# create .env with ANTHROPIC_API_KEY=...
uvicorn main:app --reload
```

### Frontend

```powershell
cd frontend
npm install
npm start            # http://localhost:4200
```

The Angular dev server proxies API calls via [proxy.conf.json](frontend/proxy.conf.json).

## Tests

```powershell
cd backend
pytest
```

## Configuration

Backend requires an `.env` in [backend/](backend/) with at minimum:

```
ANTHROPIC_API_KEY=sk-ant-...
```
